Общая структура сервера

Сервер MusicSense состоит из четырех основных частей: Web UI, Queue, Task processor и Audio manager.

Web UI - пользовательской интерфейс. Его основная задача - передавать запросы в вычислительный модуль и отображать результат его работы.

Взаимодействие пользовательского интерфейса с вычислительным модулем осуществляется средствами объекта Queue. Операция индексации музыкального трека очень ресурсоемка. Ее одновременным выполнением может заниматься сильно ограниченное количество потоков, поэтому все запросы становятся в очередь.

Из очереди запросы попадают в распределительный модуль - Task processor. В зависимости от типа, запрос направляется дальше.

Вычислительный модуль

Самый распространенный тип запроса - запрос на поиск похожих треков. Его обработкой занимается ядро приложения - вычислительный модуль(Audio manager). В свою очередь он состоит из трёх частей: Storage, Indexer, Recommendation system.

Сначала загруженный трек попадает в хранилище(Storage) - высокоуровневый объект, представляющий собой словарь [трек] -> [набор описаний].

Затем индексатор(Indexer) вычисляет и присваивает записи уникальный вектор признаков, основываясь на ее спектрограмме.

После этого рекомендательный модуль сопоставляет набор признаков загруженного файла с другими треками, находящимися в хранилище и формирует список похожих музыкальных произведений, который обратно по цепочке передается на уровень пользовательского интерфейса.
